## TL;DR

A surprising number of CoT problems can be answered correctly **before** the model
finishes its scratchpad. We train a lightweight "early-exit" head that decides, after
each step, whether the answer is already settled. The result is **40% fewer
generated tokens on average**, with **no measurable MMLU regression**.

## How it works

We add a single linear probe over the residual stream after each reasoning step. It
predicts the entropy of the would-be final answer. When entropy crosses a threshold,
generation halts and emits the answer token directly.

## Where it shines

| Benchmark | Token savings | Δ accuracy |
|:----------|:-------------:|:----------:|
| MMLU | 41% | +0.1 pp |
| GSM-8K | 33% | -0.2 pp |
| HumanEval | 12% | 0.0 pp |
| ARC-Challenge | 47% | +0.3 pp |

Math problems benefit least — the answer rarely "settles early" on multi-step arithmetic.

## Practical recipe

Train the probe on 50k self-generated rollouts. Tune one threshold per task family.
The probe is so small (∼500K params) that it adds negligible inference cost.
